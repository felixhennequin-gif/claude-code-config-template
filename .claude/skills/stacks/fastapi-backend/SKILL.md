## Route handlers

- Keep handlers thin — validate input (Pydantic), call service, return response
- Use `Depends()` for shared logic (auth, DB session, pagination)
- Type all parameters and return types — FastAPI uses these for OpenAPI docs
- Use `status_code=` parameter, not manual `JSONResponse` unless you need custom headers

## SQLAlchemy

- Always use async sessions: `async with async_session() as session`
- Use `select()` style (2.0 API), not legacy `session.query()`
- Eager load with `selectinload()` or `joinedload()` — never trigger lazy loads in route handlers
- Use `Mapped[]` type annotations on all model fields

## Alembic

- One migration per logical change — don't batch unrelated schema changes
- Always review autogenerated migrations before applying
- Name migrations descriptively: `add_user_email_index`, not `update`
- Test migrations both up and down: `alembic upgrade head && alembic downgrade -1`

## Pydantic

- Separate `Create`, `Update`, and `Response` schemas — don't reuse one model for everything
- Use `model_config = ConfigDict(from_attributes=True)` for ORM integration
- Validate at the boundary (route handler), trust validated data in service layer

## Testing

- Use `httpx.AsyncClient` with `app` for integration tests
- Use `pytest-asyncio` with `scope="function"` — don't share DB state between tests
- Factory functions for test data, not fixtures with side effects
- Test the service layer directly for unit tests, routes for integration tests

## Anti-patterns

- ❌ `db.execute(text("SELECT ..."))` for standard queries — use the ORM
- ❌ Business logic in `Depends()` — dependencies are for injection, not orchestration
- ❌ Catching broad exceptions in route handlers — let FastAPI's exception handlers work
- ❌ Synchronous DB calls — they block the event loop
