# Apply session-wrap proposals
# Usage: /wrap-apply [path-to-wrap-file]

Apply proposals the user has checked `[x]` in a file generated by `/wrap`.

**Rule: only `[x]` proposals.** Skip `[ ]` and any ambiguous variants. The checklist is the contract.

**Rule: no partial-apply.** A proposal's diff either lands cleanly or is skipped whole. Never patch half a proposal.

**Rule: no commit.** Show the diff summary at the end and stop. The user decides what to commit.

## Steps

1. **Locate the file.** If no argument, list the 5 most recent files in `~/.claude/template-proposals/` and ask the user to pick. Do not guess.

2. **Parse proposals.** Each proposal starts with `### [ ]`, `### [x]`, or `### [applied YYYY-MM-DD]`. Only `[x]` proposals are considered.

3. **Validate each `[x]` proposal:**
   - Target must be inside `.claude/**`, `CLAUDE.md`, `CLAUDE.local.md`, or `CHANGELOG.md`. Otherwise skip: "out of scope".
   - If modification: target file must exist. Otherwise skip: "target missing".
   - If creation: if target exists, ask the user before overwriting. Never overwrite silently.
   - Dry-run the patch. If context has drifted, skip: "context drift, manual merge needed".

4. **Apply clean proposals.** Direct file edit or equivalent. All-or-nothing per proposal.

5. **Handle CHANGELOG specially.** If the CHANGELOG proposal was checked:
   - Collect CHANGELOG entries only from proposals that were actually applied in this run.
   - Regenerate the CHANGELOG diff from that subset.
   - If `CHANGELOG.md` doesn't exist, create it from the Keep a Changelog 1.1.0 template with entries under `[Unreleased]`.

6. **Mark the wrap file.** Replace each successfully-applied `[x]` with `[applied YYYY-MM-DD]`. Leave `[ ]` untouched. Leave skipped `[x]` untouched and append `<!-- skipped: <reason> -->` on the next line. The file becomes a self-documenting audit trail.

7. **Report.** Applied count, skipped count (with reasons), files touched. "Not committed. Review with `git diff` then commit yourself." Stop.

## Never

- Never commit, push, or create a branch.
- Never edit files outside `.claude/**`, `CLAUDE.md`, `CLAUDE.local.md`, `CHANGELOG.md`.
- Never merge two proposals into one apply operation.
- Never re-prompt for confirmation on each proposal — the `[x]` is the confirmation.
